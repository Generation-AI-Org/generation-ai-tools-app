## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding (`px-1`, `py-0.5`) |
| sm | 8px | Tight element spacing, gap between badge and title |
| md | 16px | Default element spacing, panel padding |
| lg | 24px | Section padding inside chat panel |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets (search button, theme toggle) are minimum 44√ó44px ‚Äî matches existing `min-w-[44px] min-h-[44px]` pattern in AppShell.tsx.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 500 (medium) | 1.5 |
| Label | 12px (text-xs) | 500 (medium) | 1.4 |

Source: Extracted from `ChatPanel.tsx` header (`text-sm font-semibold`) and `Badge.tsx` (`text-[11px] font-medium`). No heading/display sizes needed for this phase ‚Äî Phase 4 only modifies the ChatPanel header and adds an inline teaser; no new page-level headings.

## Color

Both light and dark values are declared. Use CSS variables exclusively ‚Äî never hardcode hex in new Phase 4 components.

### Dark Mode (`--bg: #141414`)

| Role | CSS Variable | Value | Usage |
|------|-------------|-------|-------|
| Dominant (60%) | `--bg` | `#141414` | App background, chat panel background |
| Secondary (30%) | `--bg-card` / `--bg-elevated` | `#1C1C1C` / `#222222` | Cards, elevated surfaces |
| Accent (10%) | `--accent` | `#CEFF32` | Member badge background-tint, badge text, badge border |
| Destructive | ‚Äî | not used in this phase | ‚Äî |

### Light Mode (`.light` class on `<html>`)

| Role | CSS Variable | Value | Usage |
|------|-------------|-------|-------|
| Dominant (60%) | `--bg` | `#FAF7F8` | App background |
| Secondary (30%) | `--bg-card` / `--bg-elevated` | `#FFFFFF` / `#F5EEF0` | Cards |
| Accent (10%) | `--accent` | `#D91040` | Member badge background-tint, badge text, badge border |
| Destructive | ‚Äî | not used in this phase | ‚Äî |

**Accent reserved for (exhaustive list for Phase 4):**
- Member badge background: `bg-[var(--accent-soft)]` (`rgba(accent, 0.15)` dark / `rgba(accent, 0.08)` light)
- Member badge text: `text-[var(--accent)]`
- Member badge border: `border-[var(--accent)]/25`
- Login teaser link underline / hover color: `text-[var(--accent)]`

**Do NOT use accent for:** the teaser container background, the teaser body text, or any new structural element.

Source: `app/globals.css` (full token set), `components/ui/Badge.tsx` (accent-soft pattern), CONTEXT.md D-08.

## Component Inventory

New components introduced in Phase 4:

### 1. Member Badge (inline, no new file needed)

Rendered inside `ChatPanel.tsx` header, next to "GenAI Assistent" title. Replaces nothing ‚Äî additive.

```
[GenAI Assistent]  [Member]      ‚Üê badge inline-right of title, same row
Findet die richtigen Tools...
```

**Spec:**
- Element: `<span>`
- Position: `inline-flex`, placed after the `<p>` title text in the header flex row
- Padding: `px-2 py-0.5` (8px / 2px ‚Äî xs/xs scale)
- Border radius: `rounded-full`
- Font: `text-xs font-medium tracking-wide`
- Colors: `bg-[var(--accent-soft)] text-[var(--accent)] border border-[var(--accent)]/25`
- Label text: "Member" (not "Pro" ‚Äî see Copywriting section)
- Visibility: only when `mode === 'member'`
- Dark/Light: inherits from CSS variables ‚Äî no conditional class needed

Matches existing `Badge.tsx` free/open_source variant pattern exactly.

### 2. Login Teaser (inline, inside ChatPanel empty state)

Shown only when `mode === 'public'` AND the chat is in the empty state. Rendered below the existing `<QuickActions>` component. Not a modal, not a full-width banner ‚Äî subtle, below the fold of the welcome area.

**Spec:**
- Container: `<div className="mt-auto px-4 pb-4">`
- Padding: `px-4 pb-4` (16px / 16px)
- Background: `bg-[var(--bg-elevated)]` ‚Äî uses secondary surface, not accent
- Border: `border border-[var(--border)] rounded-xl`
- Layout: single row, `flex items-center gap-3`
- Icon: 16√ó16px inline SVG lock icon, `text-[var(--text-muted)]` ‚Äî no library import
- Text: 12px, `text-[var(--text-secondary)]` (not full `--text` ‚Äî intentionally muted)
- Link: `<a href="https://generation-ai.org" target="_blank">` with `text-[var(--accent)] hover:underline font-medium`
- No close/dismiss button ‚Äî it stays visible when chat is empty
- Never shown when `mode === 'member'`

### 3. AuthProvider (no visual output)

`components/AuthProvider.tsx` ‚Äî context only, no rendered UI. Not spec'd here beyond: it must not introduce any flash of content. The `initialUser` server-prop pattern (RESEARCH.md Pattern 3) eliminates loading skeletons ‚Äî no spinner state needed for auth in this phase.

## Interaction Contract

### Auth State Transitions

| From | To | Visual change |
|------|----|---------------|
| Loading (SSR) | V1 (no session) | No flash ‚Äî `initialUser` resolved server-side |
| Loading (SSR) | V2 (session valid) | No flash ‚Äî `initialUser` resolved server-side |
| V2 session active | V2 session expired | Silent mode switch from 'member' to 'public'; badge disappears; no toast |
| Any | Page reload | Same state restored instantly (proxy.ts refresh) |

### Member Badge Behavior

- Visible: immediately on page load if `mode === 'member'`
- Hidden: immediately if `mode === 'public'`
- No animation on appear/disappear ‚Äî functional indicator only
- Does not respond to click ‚Äî not interactive

### Login Teaser Behavior

- Visible: only in empty chat state AND `mode === 'public'`
- Disappears: as soon as user sends first message (empty state exits) ‚Äî not sticky during conversation
- Link opens in new tab (`target="_blank" rel="noopener noreferrer"`)
- No tracking parameters on the URL
