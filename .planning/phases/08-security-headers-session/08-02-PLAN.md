Erstelle lib/cors.ts mit:

1. ALLOWED_ORIGINS Array:
   - 'https://tools.generation-ai.org'
   - 'https://generation-ai.org'
   - 'http://localhost:3000' (nur wenn NODE_ENV === 'development')
   Filter null values mit `.filter(Boolean)`

2. `corsHeaders(origin: string | null): HeadersInit` Function:
   - Wenn origin in ALLOWED_ORIGINS: Return CORS Headers
     - 'Access-Control-Allow-Origin': origin
     - 'Access-Control-Allow-Methods': 'GET, POST, OPTIONS'
     - 'Access-Control-Allow-Headers': 'Content-Type, Authorization'
     - 'Access-Control-Max-Age': '86400'
   - Wenn origin nicht erlaubt: Return {} (keine CORS Headers)

3. `handlePreflight(request: Request): Response | null` Function:
   - Wenn request.method === 'OPTIONS':
     - Origin aus Headers holen
     - Response mit Status 204 + corsHeaders(origin) returnen
   - Sonst null (kein Preflight)

Export beide Functions.

<task type="auto">
  <name>Task 2: CSRF Origin Validation Library</name>
  <files>lib/csrf.ts</files>
  <action>
Erstelle lib/csrf.ts mit:

1. `validateOrigin(request: Request): boolean` Function:
   - Origin Header holen: `request.headers.get('origin')`
   - Host Header holen: `request.headers.get('host') || request.headers.get('x-forwarded-host')`
   
   - Wenn kein Origin UND kein Host: return true (same-origin request oder server-to-server)
   - Wenn Origin aber kein Host: return false (suspicious)
   
   - Try/Catch um Origin zu parsen:
     - `const originUrl = new URL(origin)`
     - return `originUrl.host === host`
   - Bei Error: return false

2. Kommentar hinzufuegen der erklaert:
   - Browser fuegen Origin Header nur bei Cross-Origin Requests hinzu
   - Same-Origin Requests haben keinen Origin Header
   - SameSite Cookies bieten zusaetzlichen Schutz

Export validateOrigin.
  </action>
  <verify>
    <automated>grep -n "validateOrigin\|origin\|host" lib/csrf.ts</automated>
  </verify>
  <done>CSRF Helper existiert mit Origin vs Host Validation</done>
</task>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External Origin -> API | Cross-Origin Requests von potentiell bösartigen Sites |
| Browser -> API | State-changing POST Requests die CSRF-anfaellig sind |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Tampering | CSRF Attack | mitigate | Origin-Check validiert dass Request von trusted origin kommt |
| T-08-06 | Information Disclosure | CORS Bypass | mitigate | Nur whitelisted Origins bekommen CORS Headers |
| T-08-07 | Elevation of Privilege | Unauthorized API Access | mitigate | CORS + CSRF zusammen schraenken API-Zugriff ein |
</threat_model>
